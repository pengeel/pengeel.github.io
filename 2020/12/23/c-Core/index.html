语法：数据类型 &变量名 = 原变量名;

2.引用的注意事项

1.引用必须初始化

2.引用初始化后不可更改

3.引用做函数参数

1.引用做函数参数时相当于 给传入的值起别名这样就是在操作传入的值

2.可以利用引用达到指针的效果

4.引用做函数返回值

1.不要返回局部变量的引用 因为局部变量在函数走完之后 做一次保留后就要销毁

2.如果函数返回的是引用 那么函数调用可以作为表达式左值

5.引用的本质

*1.引用的本质是指针常量 int * const ref = &b ==> int& ref = a*

2.当使用引用时 编译器自动转换为指针常量

3.因此 引用一旦初始化不可更改指向

6.常量引用

1.常量引用只要用来修饰形参 以防实参被改变

2.常量引用使得引用不能改变指向不能改变内容

    四、类和对象

4.1 封装

4.1.1 封装的意义

  1. 将属性和行为作为整体
  2. 并给予权限控制

注意：行为可以对属性赋值 类中的属性和行为统称为类的成员

**2.**设置访问权限

  1. Public 公共权限 成员在 改类和类外都可以访问
  2. Private 私有权限 成员 仅在该类中可访问 实例化后也不可访问私有成员
  3. Protect 保护权限 该类和其子类可访问 其他类无法访问

4.1.2stuct和class的区别

  1. class 默认的访问权限是私有(Private)
  2. struct 默认的访问权限是共有(Public)

4.1.3成员属性设置为私有

  1. 可以控制访问权限

  2. 对于写操作可以验证数据的有效性

    4.2 对象的初始化和清理

4.2.1 构造函数和析构函数

4.2.1.1

  1. 构造函数和析构函数没有返回值 也不写void
  2. 构造函数 析构函数是自动调用且只调用一次的
  3. 如不写 系统会自动创建空的构造和析构函数

构造函数

  1. 对象在建立时 会执行
  2. 可以进行重载 可以带参数

析构函数

  1. 对象在销毁时会执行
  2. 不可进行重载 不能有参数

4.2.2构造函数的分类及其调用

分类方式

  1. 有参 （拷贝构造函数）
  2. 无参

调用

  1. 括号法

    1. Person p1;默认构造
    2. Person p2(10);有参
    3. Person p3(p2)拷贝

  2. 显示法

    1. Person p4 = Person(10);
    2. Person p5 = Person(p4);
    3. Person(10);匿名对象 前面执行你玩下一行之前将回收

    注意：不要使用拷贝构造函数初始化匿名对象

  3. 隐式法

    1. Person p6 = 10;//调用有参
    2. Person p7 = p6;

4.2.3 拷贝构造函数的调用时机

三种情况

  1. 使用一个已经创建完毕的对象来初始化一个新的对象
  2. 值传递的方式给函数参数传值
  3. 值方式是返回局部对象

2.给函数传值相当于 复制一份p的值将形参给传入这是调用拷贝构造函数

3：返回局部对象时 也是重新创建新的对象 这是需要调用拷贝构造函数

4.2.4深拷贝和浅拷贝

浅拷贝

  1. 在不写拷贝构造函数时 系统默认提供浅拷贝构造函数
  2. 浅拷贝构造函数对于指针型的变量来说 仅赋值了地址并没有重新开辟新空间 以此对析构函数带来了重复释放引用的问题

深拷贝

  1. 深拷贝构造函数需要自己写
  2. 深拷贝构造函数在处理指针型变量的克隆时 会重新申请一片内存并让指针指向他 解决重复释放问题

4.2.5初始化列表

具体作用为止 目前可以代替构造函数的赋值语句

4.2.6 类对象作为类的对象

构造函数顺序

  1. 先构造所包含的对象 再调用自身的构造函数

析构函数的对象

  1. 先析构自身 再析构所包含对象的析构 与构造函数顺序相反

4.2.7静态成员

在成员变量和成员函数前加上 static关键字则称为静态成员

静态成员分为：静态成员变量和静态成员函数

静态成员变量

  1. 所有对象共有一份数据
  2. 在编译阶段分配内存
  3. 类内声明 类外初始化
  4. 静态成员变量也能有访问权限

静态成员函数

  1. 所有对象共享一个函数
  2. 静态成员函数只能访问静态的成员变量
  3. 静态成员函数也具有访问权限 private 外部也访问不到

4.3.1 成员变量和成员函数分开存储

空对象占一个字节的空间 为了和其他的空对象做区别

注意：只有非静态成员变量属于类的对象 的 其他的都不属于

4.3.2 This指针

前面说到 非静态函数不特别的属于某一个类的对象 但该函数内部可以修改 属于某个类的对象的非静态成员变量 那么怎么去区分呢

使用this指针 —>this指针指向被调用函数所属的对象

访问成员函数

  1. 空指针可以访问 没有使用非静态成员变量的函数的
  2. 可以访问 含静态成员变量的函数
  3. 一定可以访问 静态成员函数

4.3.4 const 修饰成员函数

常函数

  • ​ 成员函数加const后我们称之为常函数
  • 常函数内不可修改成员属性
  • 成员属性声明加关键字mutable后，在常函数汇总可修改

本质：修饰的是this执政 使指针指向的值也不可修改

常对象

  • 声明对象是加const关键字
  • 常对象只能调用常函数

const面向 “是否可修改”

static面向 “作用域”

4.4 友元

让一些特殊的类或者函数可以访问 其他类中的私有属性

关键字：friend

4.4.1 全局函数做友元

将需要访问该类私有字段的方法 整体声明在该类中并加入friend关键字即可

4.2 类做友元

将需要访问其他类私有字段的类 class [类名]放入需要访问的类的成员中并加入 friend关键字

4.4.3 成员函数做友元

A类中某个成员函数visit想要访问B类中的私有成员

那么 在B类中 需要A 成员函数visit的声明并加上friend关键字

怎样证明是A类的成员函数 A::visit;

friend [int] A::visit();

4.5 运算符重载

解决两个自定义类型的 加减乘除

声明方式：operator[+，-，*，/]

**注意： **

  1. 运算符也可以重载
  2. 不能滥用运算符重载
  3. 内置数据类型不能重载

4.5.1 加号运算符重载

声明（函数名）：operator+

作用：可以让自定义类型做加法

4.5.2 左移运算符重载

声明（函数名）: operator<<

作用：可以直接输出自定义类型 例如 cout << p <<endl;

4.5.3 自增运算符重载

占位符(int)可以区别两种自增运算符！！

  • 前置递增运算符：[MyClass&] operator<<(int)
  • 后置递增运算符[MyClass] operator<<(int)

注意：前置递增运算符 需要返回本身于是需要返回引用

4.5.4 赋值运算符重载

C++至少给一个类添加四个函数

  • 默认构造函数
  • 默认析构函数
  • 默认拷贝函数 浅拷贝
  • 赋值运算符 poreator+

编译器 默认的对象的=号重载 是浅拷贝 那么在析构是会产生 重复释放的问题 所以使用重载=号运算符进行深拷贝

五、继承

5.1 继承语法

语法规则： class [类名] : [继承方式] [类名]

继承的作用： 将各个类相同的代码集合起来 减少代码冗余

派生类内容：

  • 包含从父类继承下来的方法属性等
  • 自己特有的方法属性等

5.2 继承方式

私有权限可以继承 但无权限访问

继承方式有三种

  • 公共继承
  • 私有继承
  • 保护继承

公共继承：父类所有非静态属性方法的访问权限不变 子类继承时得到的这些属性跟父类中定义的属性一样

保护继承：父类所有非静态属性方法高于保护权限的 都将在子类降级为保护权限 其他不必那

私有继承：比私有访问权限高的都将降级为私有访问权限

5.3 继承的对象模型

如何查看对象模型：运行vs开发者工具cl /d1 reportSingleClassLayout[类名] [所属文件名]

5.4 继承的构造和析构顺序

构造的顺序： 先构造父类 在构造子类

析构顺序：析构时 先析构子类再析构父类与构造相反

5.5 继承中同名属性和函数子父类访问

同名成员属性

若父类(Base)存在和子类(Son)的同名属性例如 m_A;

那么创建子类的对象时访问的是 子类的属性

如果想要访问父类的同名的成员需要加上作用域 s.Base :: m_A

同名成员函数

若子类出现和父类 同名的成员函数 则子类对象 会将父类同名的成员函数全部隐藏 普通.访问是无法访问的 这时的访问需要加上作用域

5.6 继承中同名静态成员的子父类访问

和继承中 非静态的同名属性 和同名方法 处理方式一样

六、多态

6.1 多态基本语法 虚函数派生类

多态分为两种

  • 静态多态：运算符重载和函数重载 编译阶段确定地址
  • 动态多态：派生类和虚函数实现多态 在运行时确定地址

virtual关键字放在函数前 表示函数地址晚绑定 这样当继承（Cat类继承Animal类 且其中有同名函数）中有同名函数时 父类对象animal 指向 子类对象cat 时 是在程序运行时根据实际类型执行确定的函数。

6.2 多态的原理剖析

若存在继承

  1. 不存在函数重写时 Cat类 继承了Animal类的虚函数表 和&Animal::speak

  2. 存在函数重写时

    猫的虚函数指针会覆盖继承下来的&Animal::speak

    猫的虚函数表为 Cat::speak

根据动态多态 也就是运行时查找函数地址 可知在运行时查找实际类型Cat的虚函数表得到猫自己的speak函数

6.3 开发原则

开闭原则：对扩展开放，对修改关

6.4 纯虚函数和抽象类

语法：virtual 返回值类型 函数名 (参数列表) = 0 ；

含纯虚函数的类为抽象类 子类必须重写父类的纯虚函数否则子类也为抽象类

注意：抽象类无法实例化对象

6.5 虚析构和纯虚析构

目的：由于父类指针指向子类对象时 销毁时无法调用子类的析构那么实际上是造成了子类堆区内容不发释放

虚析构和纯虚析构共性

  • 解决父类指针释放子类对象
  • 都需要具体的函数来实现

虚析构和纯虚析构的区别

  1. 表示区别

  2. 作用区别
