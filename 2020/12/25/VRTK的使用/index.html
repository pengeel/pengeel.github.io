VRTK的使用

第一部分：绑定平台以及相关配置

1.1 创建两个空物体[SDKManager]和[VRTK_Script]

1.1.1 SDKManager

  • 存放各个平台的SDK并为他们使用VTRK_SDK_SetUp组件

  • 创建完成后 给SDKManager放置VRTK_SDK_Manager组件

  • 点击SDKManager_Manager的Setups的Auto Poplate使相关平台的SDK按顺序注入以便运行时的查找

1.1.2 VRTK_Script

相关的脚本挂载在上面并在SDKManager上挂载LeftController和RightController

第二部分：事件检测

基础： 在LeftController和RightController上挂载VRTK_Controller Events 这样当手柄触发了一些例如Pressed Touched等事件时会通知Events类当中的相关event 那么该event将会依次调用挂载的方法

结论：我们在开发中仅需要另外写一个脚本 挂载对该事件的处理方法即可

第三部分：光标指针

如何触发：在LeftController和RightController(挂载脚本的物体)上挂载 VRTK_Pointer这样根据Activation Button的方式就可以触发

如何显示：根据脚本Pointer Renderer 来设置发射射线的形状

显示效果：根据脚本Point Renderer 来设置发射射线的 形状（根据目标可达和不可达）

Custom Tracer 是射线的 线

Custom Cursor 是射线的抵达目标的 标

Valid Location Object 是可达目标时 标的附加显示(显示一段光束)

Invalid Location Object 是不可达

如何传送到目标位置：在Body上挂载上VRTK_Height Adjust组件

如何使特定的区域不可达

在Body上挂载可达物体的策略 根据所选的类型以及他的名字 检测是否可达

如何使角色不能移动到角落 以避免头盔穿透墙壁 (与墙壁保持指定区域)

在手的控制器上加入VRTK_Play Area Cursor脚本 并在VRTK_Bezier Pointer上的 Playarea Cursor挂载上

在Cursor游标脚本上可以更改 游标大小以及是否处理碰撞检测 如果处理 则当区域和某物体产生碰撞检测(法线非垂直)时为不可达区域

如何使射线忽略相应的层 而使射线可以进入到触发器中 而不是被阻挡

自定义该射线规则

可以使射线忽略某些层

因为玩家区域存在处理碰撞器行为的操作 那么玩家区域是不能到 某些区域的 那么如果我们需要去其中的某些区域怎么做：可以再玩家区域的目标检测规则中添加 需要忽略的规则

第四部分：拾取

配置：拾取需要手控制器上挂载 可触摸和可抓取组件、被抓取物体需要挂载可交互物体组件

注意：触摸是必须的 先要有触摸才能有别的

抛的乘积器可改变抛的乘积 少力高抛

第六部分：UI

1、2DUI

将Cavas的渲染模式设置为Screen Space- Camera 注意：Screen Space-Overlay不能用于VR设备

使用专门的摄像机 去渲染2DUI 避免UI相机看到其他的东西 我们新增一个layer让2DUI使用该layer使相机的culling mask 作用于该layer

还需要将UI摄像机的深度值设置的比 主摄像机的大

2、3DUI

新建用于存放3DUI的Canvas 将渲染模式设置为World Space 再将Cancas的Scale 缩小 因为1像素在3D世界代表着1M 因此缩小100倍为0.01

3、3DUI交互

在手上创建子物体(用于调整射线的方向) 并挂载VRTK_UI Pointer

在画布(Canvas)上挂载VRTK_UI Canvas以此就可以实现检测 配合发射线的VRTK_Pointer 和渲染器组件就可以看见射线
